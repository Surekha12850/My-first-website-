The TDD Cycle

TDD follows a simple, iterative process known as the Red-Green-Refactor cycle:
Red: Write a failing test for a specific functionality.
Green: Write the minimum code required to make the test pass.
Refactor: Improve the code's design while ensuring all tests still pass. This cycle is repeated for each new feature or functionality
